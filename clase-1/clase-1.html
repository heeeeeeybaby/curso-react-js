Premisas de React

Primera Premisa: Virtual DOM

Es una representación virtual de la interfaz de usuario que se mantiene en memoria y en sintonía con el DOM Real

Segunda premisa: Flujo de datos unidireccional

Cuando genero un evento en una aplicación con Vanilla JS, debe bajar el dato hacia el nodo y luego subir cuando se genera un evento.
Con React JS, accedo directamente al nodo que maneja los elementos que estoy trabajando y no al nodo raíz.

Ejemplo: Si trabajo en un carrito: El botón carrito involucrará sólo a los elementos que se afectan directamente y modifico sólamente el DOM del carrito

Proceso

  • React ejecuta un algoritmo de drifting que identifica lo que ha cambiado
  • EL segundo paso es la reconciliación, donde se actualiza el DOM con lso resultados de diff
  • Esto no se programa, sucede automáticamente con el framework

¿Qué es NODE js?

Hace algunos años, el backend se manejaba con PHP y JAVA, y surge la idea de generar una aplicación con un sólo lenguaje, en este caso JS. El problema es que JS fue desarrollado sólo para front-end.
Node JS es un entorno de ejecución de JS, con el que puedo desarrollar una aplicación completa.

Versión LTS: Es una versión de larga duración, mientras que la versión Actual es la versión que incorpora los últimos cambios. La versión LTS es la más recomendada dado su estabilidad.

¿Qué es Node Packager Manager? (NPM)

Nodejs implica la instalación de módulos nuevos (librerias), NPM viene automáticamente con Node y se encarga de revisar este tipo de instalaciones. para instalar la última versión.

npm install -g npm@latest

Instalar React

Es necesario instalar CLI, que es la interfaz de línea de comandos. Para proceder, debes posicionarte en la carpeta donde se instalará la aplicación. Si se instala correctamente, aparecerá el mensaje: Happy hacking!, si tienes inconvenientes en la instalación, no es recomendable volver a ejecutar el comando, hay que investigar el error. npx create-react-app nombre-app

Aplicación de React

Toda aplicación de React, tiene sólo 1 HTML, donde el react, cambia y modifica ese HTML, esto se le conoce como: Single page application

¿Qué se instaló al crear la app?

Se instala la carpeta node_modules, que no se sube al repositorio.
