Clase 6 - Comunicación API

Modelo cliente-servidor

Supone un cliente conectado a un servidor mediante un PC, smartphone o laptop. Cada uno de estos dispositivos se identifica mediante un ID y se les envía la información.

  • Supone permisos de acuerdo al usuario
  • El cliente inicia solicitando información, el servidor envía la respuesta y con ello, el fin de la comunicación

Push

Es una forma de poder trabajar, donde no se solicita la información, sino que el cliente se suscribe y le envían automáticamente la información

  1. El cliente se suscribe
  2. El servidor elige el momento del inicio de la
  3. El servicio se le provee al cliente

Polling

Ejemplo de Método polling, el sitio criptoya.com, donde los valores se van actualizando y se hace una consulta al servidor cada una cantidad de minutos definido.

Requests via HTTP/S

Vienen a ayudarnos a realizar una solicitud a un servidor, nos permiten establecer un protocolo de transferencia definido por:

URL

Es la dirección web a la que se puedes conectar, que tiene varios métodos:

  • GET: Es un método de consulta de información, como por ejemplo: fetch
  • POST: Es un método de creación de contenido
  • PUT: Se utiliza para editar la data, actualizar el stock por ejemplo. Actualizar completamente un objeto.
  • PATCH: para alterar parcialmente, es decir, sólo el stock por ejemplo
  • DELETE: Para eliminar

Query Params

Está dentro de la URL, es una especificación en las consultas a un servidor, no lo utilizaremos en esta aplicación ya que lo haremos a través de ID

URL Params / Segments

Son la especificación dentro de la URL que identifican las secciones ejemplo: coderhouse.com/alumnos/asistencia

Body

Se utiliza para transferir piezas de información entre cliente y el servidor

Headers

Se usa para definir las respuestas soportadas, requeridas, información extra, cookies, etc
