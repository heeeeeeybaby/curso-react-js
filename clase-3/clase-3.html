Clase 3 - Componentes

Diseño Modular

La idea es generar pequeños módulos y luego conectarlos a la aplicación. La interfaz se ve igual, pero es más fácil detectar errores y trabajar con un equipo más grande de personas.

Dos tipos de componentes

Pueden recibir propiedades y tienen la capacidad de hacer render de un único elemento. Un componente basado en funciones es más liviano por el uso de memoria.

Componentes basados en clases

que es complejo pero completo, se utilizaba en un comienzo y funciona heredando propiedades de otras clases padre.

Componentes basados en funciones

es lo más común, y utiliza los hooks para poder tener propiedades en las funciones.

¿Cómo reciben información estos componentes?

Un componente recibe información mediante propiedades, pasando como parámetros de un componente padre a un hijo. Lo que hace que la información siga un flujo unidireccional

Componentes de Presentación

Son componentes que simplemente muestran información pero que no incorporan mayor lógica, o una funcionabilidad específico. No tienen dependencia por una base de datos, ni estado (no tienen una variable asignada, es decir algo cuyo valor cambiará)

Composición de componentes

Componente Contenedor

Es un componente que contiene a otros y proporcionarles la información de las propiedades y tienen estado, por ejemplo: Un navbar

Componentes Hijos (Children)

Un elemento hijo se le envía la información a través de un componente padre.

Implementación de los componentes

Al crear una estructura de componentes se debe crear una carpeta llamada components dentro de la carpeta src donde se ubiquen todos los componentes y luego en el archivo index.js se debe modificar

  • Para retornar más de un elemento, debo colocarlos dentro de un contenedor div, cuando ese div no es necesario, se ocupa un fragmento, que es como la etiqueta div pero vacía

Para crear un componentes se utiliza la siguiente estructura: const CartWidget = ({cantCarrito}) => {
return(
<>
<button className="btn btn-dark">Carrito</button>
<p>{cantCarrito}</p>
</>
);
}
export default CartWidget;
